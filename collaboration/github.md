# GitHub 简介

GitHub 是全球最流行的代码托管与协作平台，基于 Git 版本控制系统，帮助个人与团队管理代码、跟踪问题并协作开发。核心特点如下：

- 版本控制：通过 Git 记录代码历史，支持分支、合并和回滚。
- 协作流程：拉取请求（Pull Request）与代码审查使多人协作更透明、安全。
- Issue & 项目管理：用 Issue 追踪需求/缺陷，配合项目看板管理迭代。
- CI/CD：与 GitHub Actions 等工具集成，实现自动测试、构建与部署。
- 生态与发现：依托海量开源项目，便于搜索、Fork、学习与复用。

## 基本概念
- 仓库（Repository）：代码与历史记录的容器，可公开或私有。
- 分支（Branch）：独立的开发线路，常用 `main`/`master` 作为默认主分支。
- 提交（Commit）：一次快照式变更，包含代码差异与说明。
- 拉取请求（Pull Request）：提议将分支改动合并到目标分支，并进行审查。
- Issue：记录待办、缺陷或讨论。
- Fork：复制他人仓库到自己账号，便于独立开发。
- Star/Watch：Star 表示收藏关注，Watch 表示关注更新通知。

## 常见使用场景
- 个人项目托管与备份，展示作品集。
- 团队协作开发与代码评审。
- 参与或维护开源项目。
- 自动化测试/构建/发布（GitHub Actions）。

## 入门步骤
1) 注册账号并完善个人资料、SSH/HTTPS 凭证。
2) 创建仓库：设置名称、描述与可见性。
3) 克隆仓库：`git clone <仓库地址>`。
4) 日常开发：本地修改 -> `git add` -> `git commit` -> `git push`。
5) 提交 Pull Request 并通过代码审查合并。
6) 用 Issue 追踪需求和缺陷，必要时创建项目看板规划任务。

## 安全与最佳实践
- 为账号启用双重验证（2FA）。
- 在提交前检查敏感信息（如密钥、密码），避免上传。
- 保护主分支：启用受保护分支、必须通过审查/检查才能合并。
- 为仓库设置合适的 LICENSE，明确使用与贡献规则。
- 使用 `.gitignore` 排除构建产物和临时文件。

## 有用的扩展与集成
- GitHub Actions：CI/CD 工作流自动化。
- GitHub Pages：托管静态站点或文档。
- Dependabot：自动检查依赖安全性与更新。
- Codespaces：云端开发环境，开箱即用。
